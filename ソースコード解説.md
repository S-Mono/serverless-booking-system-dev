# ソースコード解説：画面別メソッド一覧

## 1. HomeView.vue（ホーム画面・予約カレンダー）

### データ構造
- **Menu**: メニュー情報（id, title, price, price_with_tax, duration_min, available_staff_ids, category等）
- **Staff**: スタッフ情報（id, name, display_name, roles, is_working）
- **CustomerProfile**: 顧客プロフィール（name_kana, phone_number, is_existing_customer, preferred_category）
- **ShopConfig**: 店舗設定（営業時間、時間間隔、税率）

### 主要なリアクティブ変数
- `menus`: メニューマスタ一覧
- `staffs`: スタッフ一覧
- `selectedMenus`: ユーザーが選択したメニュー（配列）
- `reservationDate`: 予約日時
- `selectedStaffId`: 選択されたスタッフID
- `availableSlots`: 予約可能な時間枠（Date配列）
- `activeTab`: 現在表示中のカテゴリ（barber/beauty/chiro）
- `lastReservation`: 前回の予約データ

### メソッド

#### `checkCustomerStatus(user)`
**目的**: ログインユーザーの顧客情報を取得し、新規/既存を判定
**処理フロー**:
1. UIDで`customers`コレクションを検索
2. 見つかれば`customerProfile`に格納、`isNewUser`を設定
3. 見つからない場合、電話番号で名寄せを試行
4. `preferred_category`があればアクティブタブを設定

#### `fetchLastReservation(userId)`
**目的**: 前回の予約データを1件取得（「いつもの」機能用）
**処理フロー**:
1. `reservations`コレクションをクエリ（customer_id, status != cancelled）
2. `created_at`降順でソート、`limit(1)`
3. `lastReservation`に保存

#### `fetchUnreadCount(userId)`
**目的**: 未読メッセージ数をカウント
**処理フロー**:
1. `messages`コレクションで`customer_id`と`is_read=false`を検索
2. `unreadCount`に件数を設定

#### `bookWithLastMenu()`
**目的**: 「いつもの」ボタンで前回のメニューを自動選択
**処理フロー**:
1. `lastReservation.menu_items`からメニュー名リストを取得
2. 現在の`menus`マスタから同じ名前のメニューを検索
3. 見つかったメニューを`selectedMenus`に設定
4. `openBookingModal()`を呼び出し

#### `fetchAvailableSlots()`
**目的**: 選択された日付・スタッフ・メニューに基づいて予約可能時間を算出
**処理フロー**:
1. `reservations`コレクションから該当日のスタッフの予約を取得
2. キャンセル済みを除外し、busy区間を配列化
3. 営業時間内を時間間隔でループ
4. 各スロットが予約所要時間を確保でき、busy区間と重複せず、過去でないかチェック
5. `availableSlots`に追加

#### `openBookingModal()`
**目的**: 予約モーダルを開く前の検証とデータ準備
**処理フロー**:
1. メニューが選択されているか確認
2. **必須チェック**: `name_kana`と`phone_number`が未登録の場合、マイページへ誘導メッセージを表示
3. `availableStaffs`から最初のスタッフを自動選択
4. 現在日時を初期値に設定
5. モーダル表示フラグON、`fetchAvailableSlots()`を呼び出し

#### `submitReservation()`
**目的**: 予約データをFirestoreに保存
**処理フロー**:
1. 入力値の検証（日時、ユーザー、スタッフ）
2. 過去日時チェック
3. 予約数上限チェック（3件まで）
4. 時間枠の重複チェック（同じスタッフが既に予約済みでないか）
5. `reservations`コレクションに予約データを追加
   - `menu_items`には`price_with_tax`（税込価格）を使用
6. `messages`コレクションに「仮予約受付」メッセージを追加
7. 成功ダイアログ表示、モーダルを閉じる

#### `toggleMenu(menu)`
選択メニューのON/OFF切り替え

#### `selectTime(time)`
時間枠クリック時に`reservationDate`を設定

#### `formatTime(date)`, `formatDateJP(dateStr)`
日時フォーマット用ヘルパー

### 算出プロパティ
- `displayedMenus`: アクティブタブに応じたメニューをフィルタ
- `totalAmount`: 選択メニューの税込合計金額
- `totalDuration`: 選択メニューの合計所要時間
- `availableStaffs`: 選択メニューを全て担当可能なスタッフをフィルタ（新規顧客の場合は`accepts_new_customer`で絞り込み）

### ライフサイクル
**onMounted**:
1. `onAuthStateChanged`でログイン状態を監視
2. ログイン中なら`checkCustomerStatus`, `fetchUnreadCount`, `fetchLastReservation`を実行
3. `menus`, `staffs`, `shop_config`をFirestoreから取得
4. メニューは`order_priority`でソート、`price_with_tax`がない場合はフォールバック計算

## 2. MyPageView.vue（マイページ）

### データ構造
- **Reservation**: 予約情報（id, start_at, menu_items, status）

### 主要なリアクティブ変数
- `reservations`: ユーザーの予約一覧
- `nameKanji`, `nameKana`, `phoneNumber`: お客様情報フォーム
- `preferredCategory`: よく利用するメニュー
- `isProfileOpen`: お客様情報の開閉状態
- `isContactFormOpen`: お問い合わせフォームの開閉状態
- `contactMessage`: お問い合わせ内容
- `isSendingContact`: 送信中フラグ

### メソッド

#### `fetchReservations(userId)`
**目的**: ログインユーザーの予約一覧を取得し、プロフィールも併せて取得
**処理フロー**:
1. `reservations`コレクションを`customer_id`で検索
2. 結果を`start_at`でソート
3. `customers`コレクションからUID優先でプロフィール取得
4. 見つからなければ電話番号で名寄せ
5. 名前・電話番号が未入力なら`isProfileOpen`をtrueに（自動で開く）

#### `saveProfile()`
**目的**: お客様情報を保存
**処理フロー**:
1. **バリデーション**:
   - カナと電話番号が必須
   - 漢字名は漢字・ひらがな・スペースのみ
   - カナは全角カタカナとスペースのみ
   - 電話番号は数字とハイフンのみ
2. `customers`コレクションに`setDoc`（merge）で保存
3. ヘッダーの名前を即座に更新（`userStore.setCustomerName`）

#### `cancelReservation(id)`
**目的**: 予約をキャンセル
**処理フロー**:
1. 確認ダイアログ表示
2. OKなら`reservations`から削除
3. **関連メッセージを更新**: `messages`コレクションで`reservation_id`が一致するものを検索し、`is_cancelled=true`に設定、タイトルに「【キャンセル済】」を追加
4. ローカルの`reservations`配列からも削除

#### `sendContactForm()`
**目的**: お問い合わせを送信
**処理フロー**:
1. ログインと内容入力のチェック
2. `customers`コレクションから顧客情報取得
3. `contacts`コレクションに問い合わせ内容を保存
4. `messages`コレクションに「お問い合わせ受付」メッセージを追加
5. 成功ダイアログ表示

#### `formatDate(ts)`
Timestamp型を日本語形式にフォーマット

### ライフサイクル
**onMounted**:
1. `onAuthStateChanged`でユーザー取得
2. ログイン中なら`fetchReservations`を呼び出し

## 5. LoginView.vue（ログイン/登録）

### 主要なリアクティブ変数
- `phoneNumber`: 電話番号入力
- `verificationCode`: 認証コード
- `step`: 'input'（電話番号入力）or 'verify'（コード入力）
- `confirmationResult`: Firebase RecaptchaVerifierの結果

### メソッド

#### `sendVerificationCode()`
**目的**: SMS認証コードを送信
**処理フロー**:
1. 電話番号の形式チェック（数字とハイフンのみ）
2. `+81`形式に変換
3. Firebase `signInWithPhoneNumber`を呼び出し
4. `confirmationResult`に保存、ステップを'verify'に変更

#### `verifyCode()`
**目的**: 認証コードを検証してログイン
**処理フロー**:
1. `confirmationResult.confirm(verificationCode)`でコード検証
2. 成功したらホーム画面にリダイレクト

### ライフサイクル
**onMounted**:
RecaptchaVerifierを初期化（invisible-recaptcha）

## まとめ

このシステムは以下の特徴があります：

1. **予約システム**:
   - 時間枠ベースで空き状況を自動計算
   - 重複チェック、過去日時チェック、予約数上限チェック
   - 前回のメニューを簡単に再予約できる「いつもの」機能

2. **顧客管理**:
   - 新規/既存を自動判定
   - 論理削除で安全に顧客データを管理
   - 電話番号ベースの名寄せ

3. **メッセージング**:
   - 予約受付・確定時に自動通知
   - 未読数のリアルタイム表示
   - 管理者から顧客への個別メッセージ送信

4. **税込価格管理**:
   - メニューに`price_with_tax`を保存
   - 税率変更時は全メニューを一括更新

5. **LINEミニアプリ対応**:
   - LIFF認証
   - セーフエリア対応（ノッチ対策）

各メソッドは適切にエラーハンドリングされ、ユーザーフィードバックも充実しています。
